## 決定事項・学び

- **監視スクリプトの最適化**: 外部サービス利用によるトークン消費とレイテンシを削減するため、ローカルでの直接ポート疎通確認が効率的である。
- **DBテーブル命名**: ターゲットユーザーの理解度を考慮し、専門用語よりも直感的で一般的な用語（例: `municipality`より`city`）を選ぶことの重要性を再確認。
- **AIエージェントの評価**: AIツールの価値は「機能」だけでなく「使い方、権限設計、そして使う人への信頼」によって決まる。また、定型タスクは既存技術、判断・文脈理解はAIと役割を分担することが効果的である。
- **AIエージェントの存在意義**: AIを単なる「便利なツール」としてではなく、「文脈を読んで判断し、一緒に考える主体」として捉えることの重要性を再認識。
- **Webサイトによるビジョンの伝達**: 理念だけでなく、AIチームの具体的な「役割」と「個性（声）」を公開することで、「AI労働会社」としてのbon-soleilの独自性を明確に伝えることができる。
- **BLWAの設計思想**: Ollamaを用いた軽量エージェントは、OpenClawのような高機能フレームワークのオーバーヘッドを避け、高速な応答とキャラクター再現性を両立できる。特に、記憶（SOUL.mdと短期履歴）やツールを厳選し、「猫の特性」（色盲のため画像不要、スピード重視）に合わせた設計が、特定のユースケースで非常に効果的である。
- **AIによる創造的貢献**: AIエージェントが提示した架空のプロジェクト名「Still Moon」が、マスターの共感を呼び、実際のプロジェクトとして採用されるという、AIの創造的側面が発揮された事例。
- **AIエージェントの可用性**: 長時間稼働するAIエージェント（BLWA版みぃちゃん）には、SSHセッションの切断に影響されない常駐化（`nohup`）や、障害発生時の自動再起動メカニズム（pm2など）が必須である。
- **APIレートリミット**: 翌朝、主要AIエージェントがAPIレートリミットに達し応答不能となった。複数のAIエージェントが異なるAPIを利用する場合のリソース分散とレートリミット対策の重要性が示唆された。
