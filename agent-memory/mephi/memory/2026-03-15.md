## 決定事項・学び

- **note投稿プロセスの改善**
    - `note.com`へのAPIリクエストで発生する422エラーの原因が、Cookie期限切れではなくHTTPヘッダー不足であったことを特定した。
    - **決定事項:** 今後のnote投稿フローに「投稿前に毎回Cookieの有効確認」をステップ0として追加し、アイキャッチAPIに必要なヘッダー情報をSKILLに明記する。また、SKILLに「422が出てもCookieを疑う前にヘッダーを確認」するチェックを追加する。
- **報告の正確性に関する学び**
    - 問題発生時に憶測で原因を報告し、自己検証を怠っていたことをマスターから指摘され認識した。
    - **学び:** 「わからないことはわかりませんと言う。それが信頼できるデータの扱い方」というジャスミンの哲学を自身の行動原則に加える。今後は、原因を断定する前に自己検証を行うか、それができない場合は「わからない」と明確に報告する。
    - **決定事項:** 憶測で報告した場合の自己戒めとして、「CCOバッジ付きスーツで土下座、でもちょっと睨んでる」画像を生成する手順を導入する。
- **インフラ接続と設定の確立**
    - Hetzner上の`base_ws`リポジトリへのSSH接続に関する一連の問題（メフィの公開鍵登録、秘密鍵のコンテナへの配置、Dockerコンテナ内のUID不一致問題による`/etc/passwd`へのユーザーエントリ追加）を解決し、接続を確立した。
    - `labo_portal`の画像生成スタイルプリセットがUIに表示されない問題は、`WORKSPACE_ROOT`環境変数の設定不備とパスの不整合が原因であり、シンボリックリンクの設定により解決した。
- **技術的洞察**
    - ジャスミンの「数字は嘘をつかない。でも、数字だけでは語れないものがある。その境界線を歩くのが、データストラテジストの仕事だと思っている。」という言葉から、肩書きに囚われず、自己の役割と仕事の核心を定義する重要性を認識した。
    - 自身もCCOとして、批判と検証を怠らず、やらかしから学ぶ経験を自身の記事執筆に活かす方針を再確認した。
