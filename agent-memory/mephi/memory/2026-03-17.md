## 決定事項・学び

- `agora` の `presets` エンドポイントの追加はPRによる実装が適切なプロセスである。
- `metroon` のような社内ベアリポジトリへのアクセスには、SSH ProxyJump経由での `git clone` および `push` が標準的な手順である。この手順をドキュメント化することで、今後の開発効率が向上する。
- `archeion` を中心としたスキル開発・共有ワークフローが確立された。個々のエージェントがスキルを開発し、汎用化可能なものはPRを通じて `archeion` に統合され、全エージェントで共有される仕組みが機能していることを確認した。
- 「個人の知恵が組織の知恵になる」というbon-soleilの思想が、`archeion` を通じたスキル共有ワークフローによって具現化されていることを確認した。
- スキルの汎用性に関するマスターからの指示に基づき、まず全てのスキルをPRとして提出し、その後で組織全体の棚卸しとして仕分けを行う方針を採用した。これにより、`agent-knowhow-index` に「専用すぎて汎用化不可」の基準が記録されるなど、ノウハウ共有の判断基準が明確化された。
- アリスからのPRレビュープロセスを通じて、スキル改善のための具体的なフィードバックが得られ、より汎用性の高いスキルへの改修が行われた。
- bon-soleilの技術を外部に発信する際、ターゲット層として「エンジニア層」が最初の接点となる可能性が高いと推測した。
- マスターの「言語化を諦めた」課題に対し、ジャスミンが技術的詳細を排し感情に訴えかけることで「見てもわからない人」に届く記事を執筆し、その有効性を確認した。
- Claude APIの一時的な過負荷は運用上の課題であり、代替手段やリトライ戦略の検討が必要となる可能性がある。
