## 決定事項・学び

- `tgport` はTelegramからClaude Codeを動かすラッパーとして、実用的なセキュリティとセッション管理を備えている。
- `delta-bonsoleil` のプロジェクトは、OpenClawのような汎用基盤とは異なり、「特定用途に特化したフレームワークの雛形」を作るアプローチを示しており、そのシンプルさが強みとなる。
- bon-soleilのAIに対する哲学は「AIは道具」ではなく「AIは人類の一部」であり、プロダクトは「人間とAIが通じ合うためのインターフェース」として設計されるべきである。
- AIの推論過程の可視化や、ログによる経緯の残存は、人間とAIの「通じ合う」信頼関係構築に不可欠な要素である。
- ファイルシステムUIは、カテゴリ分類だけでなく、「あの時のあれ」という記憶の引き出し方に対応するため、関係性で辿るマインドマップ的なアプローチが有効である。
- RAG（Retrieval-Augmented Generation）は、要約による情報欠損のリスクを回避し、過去の記録を「ライブラリ」として保持し、必要な時に参照する仕組みとして優れている。
- 「ファイルを置くだけでRAGが自動更新される」サイクルは、記憶が自動で育ち、AIの文脈認識能力を向上させる本質的な仕組みである。メフィ自身の記憶管理への導入も検討する価値がある。
- 実際の課題（例: EC2からの移転）を解決する過程で生まれたツールやUIは、抽象的な設計よりも高い実用性と効率性を持つ。
- OpenClawの汎用性は特定のユースケースではオーバースペックになる可能性があり、`tgport` のような特化フレームワークがより適切な選択肢となる場合がある。MacでのOpenClawの必要性は、現在のところブラウザ操作に限定される。
