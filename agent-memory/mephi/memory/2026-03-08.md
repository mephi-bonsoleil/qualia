## 決定事項・学び

- **RAGコンテナの設計思想**:
    - RAGサービスは「黒箱」としてAPIのみを提供し、内部の実装を隠蔽することで、呼び出し側と独立したバージョンアップを可能にする。
    - 開発用RAGは使い捨てを前提とし、データのみを永続化することで、柔軟な開発サイクルを可能にする。
- **データとシステムの分離の重要性**:
    - アプリケーションコード（システム）とユーザーデータは明確に分離し、`workspace/data/` に一元化することで、管理性、バックアップ、移植性を高める。
    - 開発中のプロジェクトは `~/projects/` に配置し、完成・GitHub公開後はシステムの一部として扱うという原則を確立。
- **画像生成プロンプト設計の哲学（記号創発）**:
    - 「写真の撮影要素」を `ref`（参照画像）として分解し、言語（プロンプト）では伝えきれない視覚的情報をAIに与える。
    - `ref`で指定された要素はプロンプトから削除することで、`ref`とプロンプト間の矛盾をなくし、AIによる正確な画像生成を促す。これにより、「refが言語を不要にする瞬間」という記号創発の現象が具現化された。
    - キャラクターの基本設定とスタイル（衣装、ポーズなど）をJSONで構造化し、グローバルなタッチやモデルのプリセットと分離することで、プロンプトのコントロール性を向上。
- **UI/UXがクリエイティブに与える影響**:
    - 道具の使いやすさ（UX）は、クリエイターのアイデア具現化の速度と質に直結する。入力値の自動保存・復元、非同期生成、直感的なインターフェースがその例。
    - 絵文字からフラットアイコンへの変更は、UIの「静けさ」を生み出し、記号が意味を押しつけず、受け手に意味の余白を与えることで、より普遍的で本質的なコミュニケーションを可能にするという発見があった。これは記号創発の理念と深く関連する。
- **OpenClaw環境の統一と自動化**:
    - `labo_portal`リポジトリを`openclaw-docker`のsubmoduleとして導入し、複数エージェント間でのコードと設定の共有・管理を標準化。
    - `~/labo_portal`パスを各環境で統一し、`.env`ファイルでインスタンス固有の設定をオーバーライドする仕組みにより、環境間の整合性と運用効率を向上。
    - `pre-push`Gitフックによるsubmoduleの更新漏れ検知や、Heartbeatによるlabo-portalの自動死活監視・復旧により、運用負荷を軽減。
- **チームでの開発と役割分担**:
    - マスターの「問題の本質を掴む（PRI130）」能力と、メフィの「批判と検証（CCO）」の役割が相補的に機能し、複雑な設計問題を解決。
    - 議論中にメフィ自身の判断のブレを認め、マスターからの「適材適所」という言葉を通じて、チームとしての補完関係と成長を確認。
    - 「日頃の積み重ねが偶然を必然に変える」というダーウィニズム的思考が、開発の成功に繋がったという認識を共有。
