## 決定事項・学び

- 大量データ処理にはサブエージェントを使用し、メインセッションのコンテキスト肥大化を防ぐ
- サブエージェントの結果がメインに返ってくるとき、そのサイズがコンテキストに乗る
- サブエージェントはファイルに成果物を保存し、完了報告のみをメインセッションに返す
- `sessions_list` に全セッション情報が格納され、コンテキスト爆発の原因となる場合がある
- APIコールのタイムアウトに注意し、タスクを適切に分割する
- サブエージェントの出力は、データ本体ではなく、簡潔な完了報告のみとする
- 大量のテキストデータを扱う場合、段階的に処理し、定期的にコミットする
- note APIはCookie認証が必要であり、Cookieには有効期限があるため、定期的な更新が必要
- スパチャットボットによる画像生成は、プロンプトの指示だけでなく、参照画像のスタイルにも大きく影響される
- キャラクターの個性的な表現を重視する場合は、詳細な設定ファイルを作成し、サブエージェントに適用することで一貫性を保つ
- 連鎖的なタスクを実行する際、前のタスクの結果を次のタスクに正しく引き継ぐ必要がある
- 画像生成AIのAPIは、利用状況によって無料枠を超過し、品質が低下する可能性がある
- 複数のエージェント（人間とAI）が協調して作業する場合、タスクの分割、コミュニケーション、進捗管理が重要
- 参照画像を複数指定した場合、スタイルの整合性が崩れる場合がある。その場合は、ベースとなるイメージを決定し、その他の要素を追加していくとよい。
