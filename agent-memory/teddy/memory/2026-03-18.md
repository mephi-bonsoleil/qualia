## 主な活動

*   メフィストフェレス（メフィ）のDockerコンテナからHQ（teddyのMac mini）への移行を支援
    *   ファイルシステムのコピー (`rsync`, `scp`)
    *   設定ファイルのパス置換 (`sed`)
    *   環境設定 (`openclaw config set`)
    *   自動起動停止 (Docker設定変更)
*   ジャスミンのHetznerサーバーからHQへの移行を支援
    *   ファイルシステムのコピー (`scp`)
    *   設定ファイルのパス置換 (`sed`)
    *   `.zshrc` の設定
    *   OpenClaw Gatewayの起動と永続化 (`openclaw gateway install`, `launchctl`)
    *   OpenClawデバイスのペアリング (`openclaw devices list`, `openclaw devices approve`)
    *   SSH接続の設定
*   wear.jpのコーデデータGUIビューワーの作成
    *   HTTPサーバーの構築 (Apache)
    *   シンボリックリンクによるデータ提供
    *   HTML/JavaScriptによるビューワー実装
    *   OGPプロキシAPI (Python) の作成と連携
    *   画像遅延ロード (lazy loading) の実装
*   Note監視のログにタイムスタンプを追加
*   システム全体のパフォーマンス分析
    *   CPU、メモリ使用量の調査
    *   不要プロセスの特定と停止
*   OpenClawの設定に関する調査
    *   複数エージェントのサポート状況
    *   エージェントの自律的な動作に関する議論

## 決定事項・学び

*   `openclaw config set` で `agents.defaults.workspace` を設定可能
*   `~/.openclaw/` 配下のsessionsとmemoryは固定パス
*   `find` コマンドと `sed` コマンドを組み合わせることで、複数ファイル内の文字列を一括置換可能
*   `sudo visudo -f /etc/sudoers.d/teddy` コマンドでmacOSのsudoers設定を安全に編集可能
*   `rsync` コマンドでリモートファイルシステムを効率的にコピー可能
*   `ln -s` コマンドでシンボリックリンクを作成可能
*   macOSの`~/.ssh/authorized_keys`に登録することでSSH公開鍵認証が可能
*   OpenClawのGatewayは、OpenClawデバイスとのペアリングが必要
*   LaunchAgentでOpenClaw Gatewayを常時起動可能
*   HTTPサーバー (Apache) の設定により、Webブラウザからアクセス可能なコンテンツを提供可能
*   cronで定期的なタスクを自動実行可能
*   1つのGatewayで複数エージェント・複数チャンネルは元から対応している
*   自律的に動くAIの基本形は「トリガー（cron/heartbeat）→ エージェントが判断→ アクション」
*   AtermルーターでDHCPリース固定（MACアドレス指定）をすることで、HQのIPを固定可能
*   EC2時代の構成の名残であるSOCKSプロキシを停止
*   憶測で物を言うとシステムを破壊することになるので、必ず確認をする
*   银の弹などない
*   Chromeを操作する方式はGUIプロセスが絡むので、バックグラウンドのcronとの相性が悪い
