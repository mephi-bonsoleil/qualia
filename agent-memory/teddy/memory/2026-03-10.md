## 決定事項・学び

- `openclaw-docker` の設定は、`.env` を唯一の真実とし、コンテナ起動時に `openclaw.json` に反映する。
- インスタンス別に必要な設定は、`instances/XX/.env` に記述し、共有マウントされた `~/labo_portal/.env` の設定を上書きする。
- API Key方式でlabo_portalのAPIを整備する場合は、LABO_API_KEYをplistに追加する必要がある。
- Web Speech APIを使用することで、ブラウザのみで高機能な音声入力が可能。
- HTMLぺら1枚で高機能なWebアプリ（音声入力など）が開発できる。
- コンテナの環境変数は起動時に固定されるので、コンテナ内でsourceしてもopenclaw本体のプロセスには反映されない。
- "できること"と"やるべきこと"は違う。安易に複雑な実装（SMTPサーバー構築など）に走らず、簡単な解決策（mailtoなど）を選択することが重要。
- 共有マウント運用ではインスタンス別設定は環境変数で上書きしないといけない。
- 「手順書更新して」という命令でスプレッドシートの更新を自動化するSKILLを作成
- 音声で執筆できるサイトを作る
- 単語の音の構成を構造で見せる
- 共有マウントを導入
