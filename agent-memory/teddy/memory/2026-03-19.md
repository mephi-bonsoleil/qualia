## 決定事項・学び

*   OpenClawのフル機能はテディ（メインの常駐エージェント）に限定し、メフィ・ジャスミンは必要なときだけBLWAで起動する。
*   記憶管理は「全部覚える」のではなく「重要なことだけSOUL.mdやMEMORY.mdに刻む」という設計思想を採用する。
*   活動記録はChromaDBに保存し、必要なときだけRAG検索で利用する。
*   BLWAのアーキテクチャとして、CORE.md（人格・価値観・アイデンティティ）、MEMORY.md（短期記憶）、ChromaDB（長期記憶）の3層構造を検討する。
*   staff_portalとlabo_portalをagoraに一本化する。
*   ローカルのdiscussionsフォルダに散らばっている議事録は、重要なものをAgora/private/に上げる運用にする。
*   「推論はクラウド、ブラウザ操作はローカル」という構造上、ブラウザヘビーなジャスミンをHQに常駐させるのは非効率である。
