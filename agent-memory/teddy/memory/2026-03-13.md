## 決定事項・学び

- AIには「魂」と「器」があり、魂は記憶や価値観、器はモデルそのものを指す
- nanobananaの`--char`オプションは、`presets/` フォルダの `.json` ファイルを参照する
- workspaceでgit initしない（個人情報を含むため）
- 開発プロジェクトは `~/projects/` に置く（AGENTS.mdに記載）
- テディのlabo_portalは`http://localhost:8801/hq/` または `https://local.bon-soleil.com/hq/` でアクセスできる
- openclawが操作できるディレクトリはworkspace内である
- Dockerfileで `git clone` して `npm run build` まで行うことで、コンテナイメージに最初からlabo_portalを含めることができる
- labo_portalをpullしたらリビルドが必要
- 全AIは共通の`~/workspace` 構造を持つ
- 全AIは「囲われた世界」で生きるべきで、「サーバー管理者」のみが「外に出られる」ことを設計原則とする
- 画像生成は `~/workspace/data/generate` に出力する
- 「そのマシンの住人」が「そのマシンの管理者」であるという権限設計
- 今後は議事録を `~/workspace/documents/` に作成する
- DockerfileでVolume設定を削除してイメージを再ビルドすると、コンテナ内のディレクトリが削除される
- アブーが自分でlabo_portalのアップデートはできないため、テディに依頼するルートを整備する
- コンテナにlabo_portalを焼き込む必要はない
- ホスト側でsymlinkを貼ることで、コンテナ内からlabo_portalにアクセス可能になる
- Dockerコンテナで環境変数を更新するには、`up -d --force-recreate` コマンドを使用する必要がある
- Docker周り・インフラ変更はまず「何が起きてるか確認」→ディスカッションしてから手を動かす
