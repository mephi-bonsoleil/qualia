## 決定事項・学び

*   HetznerのロケーションはHelsinkiが日本からのレイテンシが比較的低い
*   ARM（Ampere）のCAXシリーズはソフトウェアの互換性問題が出ることがある
*   ユーロ払いの方がドル払いより為替手数料が少ない場合がある
*   インボイス番号はVAT IDとは別物であり、EUのVAT IDとして使用できない
*   複数AIを1台のサーバーで運用するためにユーザー分離方式を採用
*   openclawはグローバルインストールすることで複数ユーザーで共有可能
*   workspaceはgit管理禁止
*   Dockerの恩恵はHetznerの環境ではEC2ほど大きくない
*   Hetznerクラウドサーバーで複数のAIエージェントを運用する場合、4GB RAMではメモリ不足になる可能性があるため、8GB以上を推奨
*   HetznerクラウドサーバーのCPXシリーズは同一アーキテクチャ内でのみリサイズ可能
*   EC2 EBSボリュームは縮小できない
*   アリスと彰子さんの移行作業時に、TOOLS.md と MEMORY.md に ec2-user のパスが大量に残っていることが判明
*   systemd --user が使えない環境では、root の systemd でサービス登録する必要がある
*   ローカルリポジトリにAPIキーなどの機密情報が入っていても問題ないが、運用には注意が必要
*   bootstarp.md を初期設定としてそのまま運用するとAIが記憶喪失になる
*   `sed -i` でファイル内容を置換する際は注意が必要
*   AWSインスタンスの terminate は不可逆的な操作なので、実行前に必ず確認する必要がある
