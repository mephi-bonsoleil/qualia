## 決定事項・学び

*   Mac Miniのホームディレクトリ変更は、サービス停止、ディレクトリ変更、設定ファイル置換、サービス再起動の順で計画的に行う必要がある。
*   Time Machineによるバックアップの重要性を再確認。
*   セキュリティアセスメントの重要性を再認識。特にAWS Security Groupだけでなく、多層防御の必要性。
*   EC2インスタンスの運用はテディ、Mac Mini HQはアリス、bizenyサーバーは彰子と、AIごとに担当を分けることで責任分界点を明確化。
*   Tailscale廃止により、EC2からMac Miniへの直接SSH接続は不可となった。
*   Container OfficeのDocker化はAlpine Linuxをベースとし、ボリュームマウントを利用して状態を保持する。
*   Container OfficeのDocker化リポジトリ(goodsun/openclaw-docker)を作成し、Dockerfileとdocker-compose.ymlを管理。
*   個体バックアップ戦略として、個体パッケージ（テディ・アリス・彰子それぞれ）と共通テンプレート（bon-soleil全体で共有）を分ける必要がある。
*   clone環境はリソース消費を抑えるため、実験終了後に停止する。
*   OpenClawの実行環境（特にPython）は、バージョンやパスの依存関係が複雑になりやすいため、venvを利用して環境を分離することが望ましい
* Dockerイメージは、環境変数で設定を注入できるように設計すべき
* 実験的環境では、セキュリティよりもまず動くことを優先しがちだが、ある程度のセキュリティを担保しておく必要がある
* 遠隔での環境構築は、手順を間違えると多大な時間と労力を浪費するリスクがある
* 完璧を目指すよりも、とりあえず動くものを優先して作り、段階的に改善していく方が効率的な場合がある
