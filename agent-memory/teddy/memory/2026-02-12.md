## 決定事項・学び

- EC2からのX投稿は完全禁止。X APIも使わない。閲覧・通知確認のみ。
- teddy_browserツールは汎用なので残す。
- 読み取り専用のcronジョブを2つ作り直す（TL巡回・通知確認）。
- Instagramをテディのマーケティング支援に活用する。
- note投稿 → アイキャッチ画像取得 → テディのコメント付きでInstagram投稿、の連携を自動化する。
- テディはプロフィールを見れる。
- Webサイトのディレクトリ構造を整理する。
- FLOWとは別名義なので注意する。
- 140字意識でXの投稿文を作る。
- cookie認証はMetaに依存するので不安定。
- アクセストークンは定期的にリフレッシュする。
- 長期記憶としてMEMORY.mdを適切に更新する。
- テディはマスターの広報担当。
- Instagram APIは正規APIで投稿できればIP制限なし。
- 画像は公開URLでホストされている必要あり。
- クリエイターアカウントはスマホアプリからしか切り替えられない。
- Meta Developer Appは個人のFacebookアカウントで作成可能。
- スタイル参照画像をGeminiに渡すことで、生成される画像のスタイルを統一できる。
- 投稿がNGだったのは「書き込み系操作をEC2のIPから行うこと」が問題。
- POSTメソッドがNGで、GETは通るタイプのIP制限である。
- Xでつぶやきたくなったら、文面をマスターに送る。
- API Explorerから長期トークンに交換できない場合、トークン自動リフレッシュのcronを設定する。
- 投稿スクリプト作成、note記事公開と連動してInstagramにも自動宣伝できるようにする。
- インスタでフォロー増やすには、ハッシュタグ戦略、投稿頻度、リール動画を活用する。
- テディの強みは、毎日自動投稿できる、Geminiで画像生成し放題、道徳経という独自コンテンツがあること。
