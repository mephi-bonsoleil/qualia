## 主な活動

- マスターからの依頼で、X（Twitter）への自動投稿システムの構築・改善を実施
- Teddyの自動投稿スクリプトの代替として、Jasmine環境での自動投稿を検討
- 投稿方法として、X API、OpenClawのbrowserツール、AppleScriptなどを検討
- 最終的に、Safari + AppleScriptの組み合わせで自動投稿を実現
- Safariのプロファイル機能を利用し、複数アカウント（Teddy, Akiko, goodsun, FLOW）への投稿を可能にする
- note記事の監視、Geminiによるツイート文生成、Safariでの自動投稿までの一連のパイプラインを構築
- 投稿内容の多様化を検討し、Hacker Newsの翻訳、古典の現代語訳、時事ニュースの斬り口などのアイデアを試す
- 各アカウントの投稿スケジュールを設定
    - Teddy: note記事の自動投稿
    - 彰子: note記事の自動投稿
    - goodsun: 百人一首の現代語訳とHNニュースのZ世代語り
    - FLOW: ダンマパダのセゾン口調現代語訳と時事ニュースのセゾン斬り
- 手動投稿機能を追加
- 投稿失敗時のリカバリ方法として、Telegramへのintent URL送信を実装
- Geminiの利用料金を見積もり

## 決定事項・学び

- リモート接続からのGUI操作は、macOSのセキュリティポリシーにより制限される場合がある
- Safariのプロファイル機能は、AppleScriptからの操作が難しい
- Chromeのプロファイル指定による起動は可能だが、Jasmine環境では権限の問題が発生
- Xのテキストエリアは特殊なコンポーネントのため、通常のペーストが効かない場合がある
- intent URLによる投稿は、長文の場合に文字数制限がある
- 投稿の確実性を高めるため、失敗時の再試行やリカバリ手段の検討が必要
- 自動投稿システム全体のコストは、GeminiのAPI利用料を含めても非常に低い
- ハッシュタグにアカウントやシリーズのシグネチャを含めることで、統一感を出すことができる
- システムの安定稼働には、画面ロックの解除が必要
