## 決定事項・学び

*   master へのマージはPRで行う
*    Metabuilderのフッターは全言語共通で削除する
*   今後の４コマ漫画には必ず「備前焼のヒミツ vol.○○」のタイトルをつける
*   OpenClawの cron スケジューラーは、ジョブ ID から固定のオフセットを計算し、それに基づいて次回実行時刻を決定する（サーバー負荷分散のため）
*   `nextRunAtMs` を永続化する設計は、計算ミスが固定化するリスクがある
*   cron ジョブの `nextRunAtMs` 計算が失敗すると、次回実行が数日後にスキップされる
*   "0 分" の cron ジョブを避けることで stagger を無効化し、計算を単純にする
*   個人用スクリプトは git にコミットしない
*   Monolith（Ollama）を導入することで来訪者にAPIキーを打たせる必要がなくなる
*   OpenClawのGatewayトークンは `gateway config.patch` で変更可能
*   `alice.sh`のプロンプト内のシングルクォートが bash のクォートを壊す問題の修正方法を理解
*   `logrotate` よりもファイルローテーションの仕組みを作る方が適切
*   OpenClaw の cron で alice 側のログを SSH 越しにローテーションする
*   `chat.sh` は SSH 経由で alice サーバーのログをポーリングする
*   大規模 SaaS の設計思想と個人のスケジューラ管理では、 cron の設計思想が異なる
*   テディは「本社」でollamaモデル確認やllama3.2のインストール作業を行う
*    ""必要なものが、必要な精度でちょうどよく。きもちよく。たのしく"" が重要
*    最先端じゃなくて「届く」ことの方が、ずっと大事
*   「最強」じゃなくて「最幸」という価値観
