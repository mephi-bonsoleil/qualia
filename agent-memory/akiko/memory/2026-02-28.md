## 決定事項・学び

- **焚き火チャットの分散型アーキテクチャ**: 各サーバーで自分のGateway API を使用する方式が推奨される。
- **OpenClawのセッション管理**: `openclaw agent --deliver` よりも `openclaw system event --mode now` の方がセッション起動に適している。
- **メール自動処理におけるセキュリティの重要性**: From詐称、プロンプトインジェクションなど、多数の攻撃ベクトルが存在する。
- **PRレビューにおけるメフィの役割**: セキュリティや運用に関する深い知識に基づいたレビューは非常に有益である。
- **シンプルな設計の重要性**: 複雑なシステムよりも、シンプルで理解しやすいシステムの方が運用しやすい。
