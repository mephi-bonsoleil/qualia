## 決定事項・学び

*   画像生成にGeminiを使用する際のプロンプトで、`must be clearly readable` を入れるとテキストが丁寧に描画される。
*   Instagram投稿は、OpenClaw API経由ではなく、Linux cronからig_post.pyを直接実行する方式が確実である。
*   スケジューラーは2系統に分離：Instagram用（ひのちゃん・彰子）とnote/X通知用（マスター）。
*   投稿フローは、ひのちゃんOK→schedule.jsonに追加→cronが自動実行、の形とする。
*   サブエージェントに判断させず、システムに組み込まないことが重要。
*   Linux cronを利用して定期的な在庫レポートを作成する。
*   4コマ漫画のネタは4koma\_topics.jsonで管理し、使用済みのものは4koma\_history.jsonで管理する。
*   UIはスマホからの利用を前提に、操作性を重視する。
*   OpenClaw cronは最小限に留め、処理負荷の高いタスクはLinux cronに移行する。
*   AIに判断させないタスクは極力AIを介さず、スクリプトで自動化する（Keep Simpleの原則）。
*    SKILL（手順書）を定義することで、タスク実行の一貫性と再現性を高める。
*   base64形式での画像アップロードは、サーバー負荷が高くなる可能性があるため、代替手段を検討する必要がある。
*   `data` ディレクトリに、JSON等のデータファイルを配置する。
*   本番環境でAPIの動作確認をする際は、キャッシュの影響に注意する。
*    UIは、ひのちゃんなどエンドユーザーに寄り添って作成すると良いものができる。
*   UIは出来る限りシンプルにし、AIエージェントへの指示は別（Telegram）で行う。
*   OpenClawのagentTurnは、負荷が高い。
*   リポジトリを分けることで、コードとデータを分離できる。
*   .envファイルでシークレット情報を管理し、Gitにコミットしないようにする。
