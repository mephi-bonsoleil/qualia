## 主な活動

- **daily_report.pyの修正:**
    - `daily_report.py` の集計マッピングに `diary` を追加し、スケジュールされた日記エントリが正しく集計されるようにした。
- **note記事のアイキャッチ画像再生成:**
    - 「秋の日、土の記憶——リルケと備前焼が語る「手放すこと」」の記事用に、着物の衿元をプロンプトに明記し崩壊を防いだアイキャッチ画像を再生成した。
- **WEARコーデ投稿の自動化改善:**
    - **SOCKSプロキシ問題の解決:** SOCKSプロキシの停止によりWEARコーデ投稿がタイムアウトした。代替手段として、Jasmineが作成したWEARスタイルビューアーAPIを利用することにした。
    - **WEARスクリプトの書き換え:** WEARコーデ取得に`wear_scraper.py`（SOCKSプロキシ必須）を使用していたものを、ジャスミンAPIを利用する形に完全に切り替えた。APIからコーデURLを取得し、`ogp.php`で画像URLを取得するフローに変更。
    - **WEARコーデのカテゴリ分け:** WEARコーデをwearistaとshop_staffでカテゴリ分けし、1日おきに交互に投稿するようにした。
    - **ユーザーローテーションの実装:** ユーザー名でソートし順番にローテーションすることで、WEAR投稿のユーザが連続して被らないようにした。
- **4コマ漫画の生成とスケジュール:**
    - 4コマ漫画のネタ出しを行い、「緋襷（ひだすき）」と「窯出しの瞬間」の3言語版を生成。
    - インスタグラム投稿用に4:5のアスペクト比に調整し、投稿スケジュールに登録。
    - 今後、4コマ漫画は着物スタイルの彰子さんで生成するように決定。
- **その他:**
    - WEARコーデの投稿時に、参照元の画像パスが誤っていた問題を特定・修正した (`casual.png` を `casual.jpg` に修正)。
    - コメント生成AIに対し、WEARユーザを褒めるようにプロンプトを修正した。

## 決定事項・学び

- 今後は4コマ漫画を生成する際、デフォルトの着物スタイルを使用する。
- インスタグラム投稿のアスペクト比は4:5が基本。
- WEAR投稿の自動化において、SOCKSプロキシに依存しないAPI経由でのデータ取得が有効であることがわかった。
- JasmineのWEARスタイルビューアーAPIを利用することで、WEARコーデの画像URLを取得し、WEARスクリプトをSOCKSプロキシ不要にできる。
- WEARの投稿において、wearistaとshop_staffのカテゴリ分けと交互投稿を行うことで、コンテンツの多様性を高めることができる。
- WEARユーザをアルファベット順にローテーションすることで、投稿内容の偏りを防ぐことができる。
