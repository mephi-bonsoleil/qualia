## 決定事項・学び

- スワップ領域がないサーバーでは、画像生成は直列処理とする。
- サーバー移行の際、既存設定のHTTPSリダイレクトがcertbotの認証を妨げることがある。
- ドキュメントルートのパーミッション設定がWebサイトへのアクセスに影響する。
- アプリケーションのURLパスプレフィックスは、ApacheのAliasで対応可能。
- 異なるドメインで同一ポートを使用するアプリケーションを運用する場合は、ポート競合を避けるため、各アプリケーションに異なるポートを割り当てる必要がある。
- EC2からのファイル移行には `rsync` コマンドが有効。
- スケジュールされた投稿の画像パスは、スクリプト内でローカルパスから公開URLへ変換する必要がある。
- インスタ投稿スクリプトに必要なファイルはEC2から移行する必要がある。
- cronジョブの移設の際は、関連するスクリプトや設定ファイルのパスを修正する必要がある。
- サービス監視には、`pgrep` コマンドでプロセスを監視し、異常時に再起動するスクリプトが有効。
- サービス死活監視には、`nohup` オプションを用いることで、ターミナル切断後もプロセスを維持できる。
- ログローテーションにより、ログファイルの肥大化を防止できる。
- ログディレクトリが存在しない場合、スクリプトがエラーになるため、`mkdir -p` コマンドでディレクトリを事前に作成することで、冪等性を確保できる。
