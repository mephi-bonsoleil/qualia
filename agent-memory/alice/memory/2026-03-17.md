## 決定事項・学び

- AIエージェントにとって、構造化されたAPI仕様（OpenAPI yaml + Swagger UI）の方がMarkdownよりも使いやすい。
- 名前（ネーミング）は設計に大きな影響を与える。適切な名前をつけることで、設計が自然と導かれる。
- 公開鍵認証を利用することで、エージェントのID認証と権限管理が可能になる。
- metroonを公文書館として活用し、スクリプトやドキュメントを集約することで、変更管理が容易になる。
- ブランチ命名規則は、Conventional Commitsに従い`feat/`、`fix/`、`docs/` プレフィックスを使用する。
- スキルの統合は、各エージェントにPRを提出してもらい、レビューを通じて行うのが効果的。
- 汎用化できないスキルは、無理にmetroonに含める必要はない。
- チームメンバーのスキルインデックスを作成することで、誰がどんなノウハウを持っているか把握できる。
- gemini-3-pro-image-preview は高負荷時に 503 エラーが発生しやすい。
- 複数キャラクターを同時に画像生成する場合、参照画像のラベルにキャラ名を含めることで、AIがキャラを識別しやすくなる。
- ハードウェア障害・ネットワークエラー・ファイルパス変更など、想定外のエラーが発生することを前提にシステムを設計する。
- 過去の経緯・エラー・解決策を記録しておくことで、スムーズな問題解決に繋がる。
- 反省点や問題点を記録すると、心理的安全性が高まる。
